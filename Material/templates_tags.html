1. O que são Templates?

Templates são arquivos HTML que o Django processa para gerar páginas dinâmicas.

Eles permitem:

  • Separar lógica de negócio da apresentação
  • Reutilizar código HTML
  • Inserir dados dinâmicos vindos das views
  • Aplicar lógica de apresentação (loops, condicionais)

Django Template Language (DTL) - linguagem simples e poderosa

2. Configurando Templates

Estrutura de Diretórios

mercado/                    # Projeto principal
├── templates/              # Templates globais
│   └── base.html
└── produtos/               # App
    └── templates/
        └── produtos/       # Namespace do app
            ├── lista.html
            └── detalhe.html
Por que usar namespace (produtos/)?
Para evitar conflitos quando apps diferentes têm templates com o mesmo nome.

Configuração no settings.py

TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [BASE_DIR / 'templates'],  # Templates globais
        'APP_DIRS': True,  # Busca templates em cada app
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

3. Herança de Templates

A herança permite criar um template base com a estrutura comum e reutilizá-lo em várias páginas.

Benefícios:

  • Reutilização de código
  • Manutenção facilitada
  • Consistência visual
  • Não repetir código (DRY - Don't Repeat Yourself)

Criando o Template Base

templates/base.html:

{% load static %}
<!DOCTYPE html>
<html lang="pt-br">
<head>
    <title>{% block title %}Sistema de Mercado{% endblock %}</title>
    <link rel="stylesheet" href="{% static 'css/style.css' %}'">
    {% block extra_css %}{% endblock %}
</head>
<body>
    <nav class="sidebar">
        <!-- Menu de navegação -->
    </nav>
    
    <main>
        {% block content %}
        <!-- Conteúdo específico aqui -->
        {% endblock %}
    </main>
    
    <script src="{% static 'js/style.js' %}"></script>
</body>
</html>

Conceitos Importantes

{% block %}

Define áreas que podem ser substituídas nos templates filhos

{% block content %}
  <!-- conteúdo -->
{% endblock %}

{% load static %}

Carrega a funcionalidade de arquivos estáticos

{% load static %}
{% static 'css/style.css' %}

Herdando do Template Base

produtos/templates/produtos/lista.html:

{% extends 'base.html' %}

{% block title %}Produtos - {{ block.super }}{% endblock %}

{% block content %}
<div class="produtos-container">
    <h2>Produtos Disponíveis</h2>
    <p>Total: {{ total_produtos }} produtos</p>
</div>
{% endblock %}
Importante: {% extends 'base.html' %} deve ser a primeira linha!

4. Passando Dados para Templates

Criando um Template Simples

Passando Dados da View

produtos/views.py:

from django.shortcuts import render

def lista_produtos(request):
    contexto = {
        'titulo': 'Lista de Produtos',
        'produtos': [
            {'nome': 'Arroz 5kg', 'preco': 25.90, 'estoque': 150},
            {'nome': 'Feijão 1kg', 'preco': 8.50, 'estoque': 200},
            {'nome': 'Açúcar 1kg', 'preco': 4.99, 'estoque': 120},
        ],
        'total_produtos': 3
    }
    return render(request, 'produtos/lista.html', contexto)

5. Tags Principais

Tag {% if %} - Condicionais

Use {% if %} para exibir conteúdo condicionalmente.

{% if condicao %}
    <p>Conteúdo exibido se verdadeiro</p>
{% endif %}

{% if usuario.is_authenticated %}
    <p>Bem-vindo, {{ usuario.nome }}!</p>
{% else %}
    <p>Por favor, faça login.</p>
{% endif %}

Tag {% if %} - Operadores

<!-- Comparação -->
{% if estoque > 10 %}
    <span>Em estoque</span>
{% endif %}

{% if estoque < 5 %}
    <span>Estoque baixo!</span>
{% endif %}

<!-- Lógicos -->
{% if idade >= 18 and habilitado %}
    <p>Pode dirigir</p>
{% endif %}

{% if promocao or desconto %}
    <span>Oferta!</span>
{% endif %}

Tag {% for %} - Loops

Use {% for %} para percorrer listas e exibir dados repetidos.

<ul>
    {% for produto in produtos %}
        <li>{{ produto.nome }} - R$ {{ produto.preco }}</li>
    {% endfor %}
</ul>

<!-- Com {% empty %} -->
<ul>
    {% for produto in produtos %}
        <li>{{ produto.nome }}</li>
    {% empty %}
        <li>Nenhum produto encontrado</li>
    {% endfor %}
</ul>

Variáveis do forloop

{% for produto in produtos %}
    <div>
        <span>{{ forloop.counter }}</span>
        <h3>{{ produto.nome }}</h3>
        
        {% if forloop.first %}
            <span>Destaque</span>
        {% endif %}
        
        {% if forloop.last %}
            <span>Último</span>
        {% endif %}
    </div>
{% endfor %}
Variáveis disponíveis: forloop.counter, forloop.counter0, forloop.first, forloop.last

Tag {% url %} - URLs Nomeadas

Use {% url %} para gerar URLs baseadas nos nomes do urls.py.

<!-- URLs simples -->
<a href="{% url 'home' %}">Dashboard</a>
<a href="{% url 'produtos:lista' %}">Ver Produtos</a>

<!-- URLs com parâmetros -->
<a href="{% url 'produtos:detalhe' produto_id=5 %}">
    Ver detalhes
</a>

<a href="{% url 'vendas:relatorio' ano=2024 mes=3 %}">
    Relatório Março/2024
</a>
Vantagem: Se você mudar a URL no urls.py, os templates são atualizados automaticamente!

6. Arquivos Estáticos

Tag {% load static %}

{% load static %}

<!-- CSS -->
<link rel="stylesheet" href="{% static 'css/style.css' %}">

<!-- JavaScript -->
<script src="{% static 'js/style.js' %}"></script>

<!-- Imagens -->
<img src="{% static 'img/logo.png' %}" alt="Logo">
<img src="{% static 'img/profile.png' %}" alt="Perfil">

Estrutura de Diretórios

mercado/
├── static/                  # Arquivos estáticos globais
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── style.js
│   └── img/
│       ├── logo.png
│       └── profile.png
└── produtos/
    └── static/
        └── produtos/        # Namespace do app
            ├── css/
            │   └── produtos.css
            └── js/
                └── produtos.js

Configuração no settings.py

# URL para acessar arquivos estáticos
STATIC_URL = '/static/'

# Diretórios adicionais de arquivos estáticos
STATICFILES_DIRS = [
    BASE_DIR / 'static',
]

# Diretório onde collectstatic reúne arquivos (produção)
STATIC_ROOT = BASE_DIR / 'staticfiles'

7. Exemplo Completo

View com Dados Fictícios

def home(request):
    contexto = {
        'page_title': 'Dashboard',
        'estatisticas': [
            {'label': 'Total de Produtos', 'value': 150},
            {'label': 'Vendas Hoje', 'value': 42},
            {'label': 'Clientes Ativos', 'value': 328},
        ],
        'produtos_destaque': [
            {'nome': 'Arroz 5kg', 'preco': 25.90, 'estoque': 150},
            {'nome': 'Feijão 1kg', 'preco': 8.50, 'estoque': 200},
        ]
    }
    return render(request, 'home.html', contexto)

Template com Herança e Tags

{% extends 'base.html' %}

{% block content %}
<div class="dashboard">
    <div class="stats-grid">
        {% for stat in estatisticas %}
        <div class="stat-card">
            <h3>{{ stat.value }}</h3>
            <p>{{ stat.label }}</p>
        </div>
        {% endfor %}
    </div>
    
    {% if produtos_destaque %}
        {% for produto in produtos_destaque %}
        <div>
            <h3>{{ produto.nome }}</h3>
            {% if produto.estoque < 10 %}
                <span>Estoque baixo!</span>
            {% endif %}
        </div>
        {% endfor %}
    {% endif %}
</div>
{% endblock %}

8. Boas Práticas

✅ Faça

  • Use herança de templates
  • Carregue {% load static %}
  • Use URLs nomeadas
  • Use {% empty %} em loops
  • Use {% if %} para condicionais

❌ Evite

  • URLs fixas/hardcoded
  • Esquecer {% load static %}
  • Lógica complexa em templates
  • Repetir código (use herança!)
  • Não usar namespace
