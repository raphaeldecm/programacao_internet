🧩 O que é um App?

No Django, o projeto é o conjunto de configurações; o App é onde a funcionalidade vive.

  • Modularidade: Separe "Blog" de "Vendas".
  • Desacoplamento: Um app não deve depender internamente de outro.
Comando: python manage.py startapp nome_do_app

🔗 Roteamento Desacoplado

O arquivo de rotas do projeto delega o tráfego para o app específico.

projeto/urls.py (Principal)

            from django.urls import path, include

            urlpatterns = [
                path('admin/', admin.site.urls),
                # Delega tudo que começa vazio para o app 'website'
                path('', include('website.urls', namespace='web')), 
            ]

🛡️ Namespaces (app_name)

Evita que duas rotas com o mesmo nome (ex: 'index') entrem em conflito.

website/urls.py (No App)

            from django.urls import path
            from . import views

            app_name = 'web' # O identificador do app

            urlpatterns = [
                path('', views.home, name='index'),
            ]

Referência no HTML: {% url 'web:index' %}

🛠️ Mão na Massa: Modularização

Objetivo: Criar um App de Portfólio

  1. Crie um novo app chamado portfolio.
  2. Crie uma view simples contato que renderiza um template.
  3. Configure o urls.py do app com app_name = 'port'.
  4. No app principal (website), crie um link no menu para a página de contato usando:
    <a href="{% url 'port:contato' %}">

⏱️ Tempo: 30 min | 💡 Dica: Não esqueça de registrar o app no settings.py!
